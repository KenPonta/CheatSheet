# CheeseSheet Requirements
# System requirements and dependencies for CheeseSheet application

# =============================================================================
# SYSTEM REQUIREMENTS
# =============================================================================

# Node.js Runtime
Node.js >= 18.0.0
npm >= 9.0.0 (or pnpm >= 8.0.0)

# Operating System Support
- macOS 10.15+ (Catalina or later)
- Ubuntu 20.04+ / Debian 11+
- Windows 10+ (with WSL2 recommended)
- Docker support on all platforms

# Memory Requirements
RAM >= 4GB (8GB recommended for development)
Disk Space >= 2GB free space

# EXTERNAL DEPENDENCIES

# Ubuntu/Debian: texlive-full or texlive-latex-base

# Docker: Included in container

# =============================================================================
# CLOUD DEPLOYMENT REQUIREMENTS
# =============================================================================

# Vercel
- Node.js 18.x runtime
- 1GB memory limit
- 50MB deployment size limit
- Serverless functions timeout: 10s (Hobby) / 60s (Pro)

# AWS
- EC2: t3.medium or larger
- Lambda: 1GB memory, 15-minute timeout
- ECS: 1 vCPU, 2GB memory minimum

# Docker Swarm/Kubernetes
- 1 CPU core, 2GB RAM per replica
- Persistent volume for LaTeX temp files
- Load balancer for multiple replicas

# API RATE LIMITS

# OpenAI API
- GPT-4: 10,000 tokens/minute (Tier 1)
- GPT-3.5-turbo: 90,000 tokens/minute (Tier 1)

# Recommended: Tier 2+ for production use

# =============================================================================
# INSTALLATION COMMANDS
# =============================================================================

# Quick Setup
git clone <repository-url>
cd cheesesheet
./setup.sh

# Manual Setup
npm install
cp .env.example .env.local
# Edit .env.local with your API keys
npm run build
npm start

# Docker Setup
docker-compose up -d

# =============================================================================
# PERFORMANCE RECOMMENDATIONS
# =============================================================================

# Development
- Use pnpm for faster installs
- Enable SWC for faster builds
- Use --max-old-space-size=4096 for large files

# Production
- Enable Redis caching
- Use CDN for static assets
- Configure proper memory limits
- Enable gzip compression
